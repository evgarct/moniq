# UI Guidelines

Moniq should read like a finance product, not an admin dashboard.

## Principles

- Prefer lists, rows, dividers, and alignment over card grids.
- Keep information dense but readable.
- Use decoration sparingly; hierarchy should come from typography and spacing.
- Reserve color for positive and negative financial states.

## Typography

- Use Inter for UI copy and JetBrains Mono for money values.
- Keep headings compact and close to the content they describe.
- Let numeric values dominate labels through size, not heavy color.
- Avoid stacking many font weights on one screen.

## Surfaces

- Use flat sections with subtle separators.
- Keep border radius small.
- Avoid heavy borders, nested boxes, and oversized pills.
- Active navigation and selected rows should use soft tonal emphasis, not loud chrome.

## Accounts UI

- The balance workspace should use a dark blue finance canvas with a narrow icon rail and a compact top bar.
- Account sections should use muted header bars with totals aligned right, followed by ledger-like rows with the amount shown under the account name.
- Savings details should stack as compact summary strips, free money emphasis, allocation rows, and register rows.
- Register and allocations should scan vertically like operational lists, with colored amount chips only for the final money value.
